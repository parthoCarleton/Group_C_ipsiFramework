int ipsi_connection_type ( char *  app_name,
char *  connection_type 
)

This is an External API use by the application to register with IPSI library and specify its role as "Caller"/"Server" along with a unique name.

Service requester Application should register itself as "Caller" and with a unique name, this will allow the application to invoke service request to desired service provider. Service provider Application should register itself as "Server" and with a unique name, this will allow the application to server any service request and allow service requester application to invoke services with the unique name.

Parameters
[in]app_nameUnique Name of the application
[in]connectionTypeSpecify the role of the application either "Caller" or "Server"
Returns
SUCCESS: The application as successfully registered itself with IPSI library
FAILURE: The application was unable to register with IPSI library , an appropriate error message will be shown on console

◆ ipsi_listen()

int ipsi_listen ( )

This is an External API that will allow Service Provider to listen for service request.

Service Provider will listen for service request that are requested by the Service Requester Application. If the service request is valid it will invoke the service that has been registered with the IPSI library with ipsi_register_function() by the Service provider application.

Note
This function will block the current thread on which it was called until service invocation was performed and then it will resume to block the current thread for listening any other service request.
To stop listening for any service request , call ipsi_close() API this will free the current thread on which ipsi_listen was operating and should exit with SUCCESS.
Returns
SUCCESS: It was able to exit Successfully
FAILURE: It was not able to exit Successfully as it was not able to able to exit from system's IPC properly

This is an External API use to register callback functions for the services offered by the Service Provider.

Service Provider should register its callback functions with a unique name. This unique service name can be used by the Service requester to invoke service request by using ipsi_method_call().

Parameters

[in]method_nameUnique name of the callback function that Service Provider application should provide to identify its services request
Returns

FAILURE: The function call was unable to register the function with IPSI library
